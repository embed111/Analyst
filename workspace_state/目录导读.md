# workspace_state 目录导读

## 1. 顶层规则
1. `workspace_state/` 顶层仅保留本文件。
2. 其他文档必须放在子目录：`core/`、`policies/`、`observations/`、`logs/`。
3. 若新增顶层文件，需在同轮迁移并记录到 `workspace_state/logs/state-change-log.md`。

## 2. 子目录导航
1. `core/`
   - `core/startup-checklist.md`
   - `core/session-snapshot.md`（历史冻结，不再维护；历史内容已转入 `.codex/memory/YYYY-MM/YYYY-MM-DD-每日记忆.md`）
   - `core/workspace-state-overview.md`
2. `policies/`
   - `policies/preference-attribution-policy.md`
3. `observations/`
   - `observations/external-input-observations.md`
4. `logs/`
   - `logs/state-change-log.md`
   - `logs/session-history-index.md`
   - `logs/session-history/`

## 3. 动态维护（归档）
1. 巡检命令：
   - `scripts/maintain-state-health.ps1`
2. 自动归档命令：
   - `scripts/maintain-state-health.ps1 -AutoArchive`
3. 默认触发条件（命中任一即归档）：
   - `user_profile/logs/thinking-patterns-change-log.md` 日期块数 >= 25；
   - `user_profile/logs/thinking-patterns-change-log.md` 行数 >= 500 或体积 >= 120 KB；
   - 距离上次偏好日志归档 >= 14 天。

## 4. 关联入口
1. 用户偏好入口：`user_profile/目录导读.md`
2. 全局规则入口：`AGENTS.md`
3. 工作连续性入口：`.codex/MEMORY.md` + `.codex/memory/<当前年月>/<当日日期>-每日记忆.md`
