# 需求详情-训练闭环页签与滚动分层约束

## 1. 主题目标与价值说明
1. 将训练闭环中的“页签固定、滚动分层、布局规则不在页面常驻展示”沉淀为正式需求口径。
2. 避免执行方或评审把原型中的布局说明误读为页面正式业务信息。
3. 保持训练闭环页面的阅读顺序稳定：先看当前任务与阶段，再通过页签查看不同信息视图。

## 2. 用户画像与使用场景
1. 用户画像：
   1. workflow 使用者：在固定窗口内查看训练创建与任务状态。
   2. 需求分析师/产品评审者：需要从原型和需求文档中分辨“业务信息”与“布局规则”。
   3. 执行工作区实现者：需要明确页签、滚动和页面文案的边界，避免实现偏差。
2. 使用场景：
   1. 用户进入 `训练中心 -> 训练闭环 -> 创建任务`，查看基础信息、首轮工作集与启动确认。
   2. 用户进入 `训练中心 -> 训练闭环 -> 任务状态`，查看当前概览、工作集变化、三轮评测与历史记录。
   3. 页面在单窗口内承载多类信息，但不允许通过页面常驻解释文案传达布局规则。

## 3. 用户旅程或关键流程
1. 用户进入训练闭环页面。
2. 页面先展示当前任务、当前阶段与主判断。
3. 用户通过主卡顶部页签切换需要查看的信息视图。
4. 若当前视图内容超出可见高度，用户在页签下方内容区滚动查看。
5. 页面始终不把“页签固定在顶部/只展开一个主标签/不再同屏摊开”等规则作为正文说明展示。

## 4. 功能需求清单
1. FR-UO-37 页签层级与滚动边界
   1. `任务状态` 的 `当前概览 / 工作集变化 / 三轮评测 / 历史记录` 与 `创建任务` 的 `基础信息 / 首轮工作集 / 启动确认` 应作为中部主卡内的视图切换页签。
   2. 页签栏必须固定在对应主卡顶部，并与下方内容滚动区形成明确分层。
   3. 桌面基线视口下，页签栏默认应完整可见，不应依赖横向或纵向滚动承载其自身内容。
   4. 内容超出可见高度时，滚动只允许发生在页签下方的内容承载区，而不是页签栏本身。
   5. 默认进入页签应为：
      1. `创建任务`：`基础信息`
      2. `任务状态`：`当前概览`
   6. 当桌面有效宽度接近 `1280px`（例如 `1920x1080` 且系统缩放 `150%`）时，页签栏应先降级为紧凑等宽布局，避免出现文字被挤压、截断后不可辨认或整项不可见。
   7. 当窄桌面导致中部高度受限时，位于页签上方的流程步骤条也应先降级为紧凑形态，避免因步骤条占高过多把主页签整体挤出首屏。
   8. 若紧凑页签与紧凑步骤条后，中部主阅读区仍无法在首屏完整展示主页签与主内容入口，应允许中间主栏升级为独立纵向滚动区；左侧任务列与右侧判定列继续保持各自独立滚动。
2. FR-UO-38 布局口径不得作为页面常驻文案
   1. 关于“页签固定在顶部、不滚动、只展开一个主标签、低优先信息通过切换进入”等布局规则，只允许写入需求文档、设计文档、实现注释或专用帮助入口。
   2. 训练闭环页面的常驻文案应优先表达业务信息、阶段状态、结果判断与后续动作，不得长期混入布局说明文字误导用户。

## 5. 非功能需求
1. 可用性：
   1. 页签栏应保持可发现性，用户进入页面后无需滚动即可识别“这里可以切换”。
   2. 当前激活页签可以使用轻量视觉强调，但不能做成高强度功能按钮。
2. 一致性：
   1. `创建任务` 与 `任务状态` 两个视图采用一致的页签层级语义与滚动分层原则。
3. 可维护性：
   1. 布局规则应只保留在文档与实现注释中，减少未来因页面文案引发的误实现和误评审。

## 6. 验收标准
1. AC-UO-35 页签固定、滚动留给内容区
   1. Given 用户进入 `训练中心 -> 训练闭环 -> 创建任务` 或 `任务状态`
   2. When 页面在桌面基线视口下首次渲染
   3. Then 页签栏固定显示在对应主卡顶部
   4. And 页签栏本身不出现承载主要内容的滚动条
   5. And 超出高度的内容只在页签下方内容区内部滚动
2. AC-UO-36 页面不展示布局规则型常驻文案
   1. Given 用户查看训练闭环页面主界面
   2. When 页面展示创建任务或任务状态的主阅读区
   3. Then 页面常驻文案不直接展示“固定在主卡顶部 / 本身不滚动 / 只展开一个主标签 / 不再同屏摊开”等布局说明
   4. And 相同规则可在需求与设计文档中追溯到正式口径
3. AC-UO-37 窄桌面下页签仍完整可辨认
   1. Given 用户在 `1920x1080` 且系统缩放 `150%` 的桌面环境中查看训练闭环页面
   2. When `创建任务` 或 `任务状态` 的中部主卡宽度被压缩到接近 `1280px` 有效宽度
   3. Then 页签栏自动切换到紧凑等宽布局或等效降级布局
   4. And `基础信息 / 首轮工作集 / 启动确认` 等页签仍完整可见、可辨认
   5. And 页面不通过页签栏滚动条解决该问题
4. AC-UO-38 窄桌面下主页签不被上方步骤条挤出首屏
   1. Given 用户在 `1920x1080` 且系统缩放 `150%` 的桌面环境中查看 `创建任务`
   2. When 中部主卡同时展示流程步骤条与主页签
   3. Then 流程步骤条先压缩为紧凑形态
   4. And `基础信息 / 首轮工作集 / 启动确认` 仍保留在主阅读区可见范围内
   5. And 页面不通过隐藏主页签或把主页签整体挤出首屏来换取收纳
5. AC-UO-39 窄桌面下中间主栏可独立滚动
   1. Given 用户在 `1920x1080` 且系统缩放 `150%` 的桌面环境中查看训练闭环页面
   2. When 中部主阅读区在首屏无法同时容纳顶部流程卡、主页签与正文入口
   3. Then 中间主栏出现独立纵向滚动条
   4. And 用户可通过该滚动条继续查看被下压的主页签与正文内容
   5. And 页面不退化为整页长滚动

## 7. 边界条件与异常处理
1. 若后续页签数量增加到桌面基线视口无法完整容纳，优先合并低频项、重构分组或引入 `更多` 入口，不优先给页签栏加滚动条。
2. 若紧凑等宽布局后仍无法完整承载全部页签，优先将低频页签降级到 `更多` 入口，而不是继续挤压字号或引入页签滚动条。
3. 若产品后续需要提供帮助说明，应进入专用帮助入口、悬浮提示或文档系统，不直接占用页面正文主阅读区。
4. 若某页签暂无数据，应展示业务空态，不得回退为布局规则说明文字。
5. 中间主栏滚动仅作为窄桌面/高缩放场景下的 fallback，不取代常规桌面下“页签固定 + 内容区滚动”的主阅读结构。
